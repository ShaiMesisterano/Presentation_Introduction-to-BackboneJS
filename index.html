Part 2

Backbone's architecture

How to find...

Features

  • MV(C) structure.
    • Client-side app structure.
    • Seperation of concerns.
    • Keeping it DRY.
  • Lazy Rendering.
    • Fast response.
    • A better user experience.
    • Happy users - more money!
  • Templates.
    • Less HTML.
    • Less DOM manipulation.
    • Less programming work.

Dependencies

  • Underscore.js
    • Data queries.
    • Templates library.
    • A replacement for ugly loops.
  • jQuery

Part 3

Backbone.Model

What is a model?

  • Set of properties.
  • Represents data.
  • Like a row in table:
    ID Source Description
    1 images/dog.jpeg Dog
  • Alternative form:
    • ID: 1
    • Source: images/dog.jpeg
    • Description: Dog

CRUD

  • Create.
  • Read.
  • Update.
  • Delete.

How to Create a model?

  • Instantiate.
  • defaults property (optional).

What can we do with a model?

  • Read it: toJSON
  • Read its field: get
  • Update its field: set
  • Delete it: clear
  • Listen to its events: on

Part 4

Backbone.Collection

What is a collection?

  • A replacement for group of models.
  • Like a table:
    ID Source Description
    1 images/dog.jpg Dog
    2 images/cat.jpg Cat
    3 images/monkey.jpg Monkey

Model vs. Collection

How to Create a collection?

  • Instantiate.
  • model property.

What can we do with a collection?

  • Read it: toJSON
  • Listen to its events: on

What can we do with a model inside a collection?

  • Create it: add
  • Read it: get
  • Update it: set
  • Delete it: clear
  • Iterate through it each

Part 5

Views

What is a view?

  • What we see.
  • Hooks up models to the DOM.
  • No Logic inside!

How to Create a view?

  • Instantiate.
  • el and tagName property.
  • id property.
  • className property.

What can we do with a view?

  • Render its element ( this.$el): render
  • Delete it: remove

How can we catch events inside a view?

  • click
  • dblclick
  • mouseenter
  • mouseleave
  • ...
